Problématique

Les vulnérabilités liées à l'utilisation de l'IA en tant qu'assistant de développement.

Méthodes de veille

Pour rester à jour et sélectionner des informations fiables, j’ai utilisé les outils suivants :

  • Feedly Agrégation et organisation des flux RSS de blogs dev & IA
  • Google Alerts Alertes personnalisées pour GitHub Copilot et IA
  • X (Twitter) Veille en temps réel, threads experts et actualités sur IA
  • Reddit /r/programming Discussions et retours d’expérience sur outils IA

Toutes les informations ont été filtrées et organisées via ces outils pour garantir leur fiabilité et pertinence.

Thème 1 : Vulnérabilités des assistants IA

OWASP Top 10

Analyse des vulnérabilités majeures applicables aux contextes où l’IA propose du code. Ce document est fondamental pour identifier les risques liés à l’usage de générateurs de code dans un environnement de production.

Thread Reddit /r/programming

Retour d’experts sur les cas de fuites de données et secrets via les suggestions Copilot. Discussion technique pragmatique, illustrant les problématiques rencontrées en entreprise.

Thread X (Twitter)

Post de spécialistes cybersécurité présentant des exemples d’injection de code malveillant via prompts IA. Ce fil est très opératif pour les scénarios d’évaluation des risques.

Thème 2 : Atténuation et bonnes pratiques

Microsoft Security Blog

Guide détaillé pour sécuriser le développement assisté par IA, avec recommandations sur la gestion des secrets, validations, audits et routines de revue de code.

Thread X (Twitter)

Astuces pratiques de professionnels pour auditer les suggestions IA avant intégration. Ce fil montre des cas concrets de mise en place de garde-fous.

Article Feedly

Synthèse d’études sur le renforcement du code généré par IA. Idéal pour structurer une méthodologie de revue et tests dans le pipeline de développement.

Thème 3 : Conformité, gouvernance et impacts

CISA

Bulletin officiel sur les obligations de gouvernance pour l’usage de l’IA dans les projets. Utile pour préparer la mise en conformité et les contrôles internes.

NIST

Cadre de sécurité solide pour l’AI, qui guide la définition de politiques et standards pour des assistants IA en développement.

X (Twitter)

Publication de l’ANSSI sur les bonnes pratiques d’audit pour les outils d’assistance au code, et points d’attention pour la sécurité des projets.
